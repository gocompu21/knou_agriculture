## 제9장. 데이터 교환 방식

### 9.1 컴퓨터 통신망의 목적

**컴퓨터 통신망**을 활용하는 주된 목적은 다음 세 가지이다.

- **자원의 공유**: 하드웨어, 소프트웨어, 데이터 등 컴퓨터 자원을 여러 사용자가 공동으로 활용한다.
- **신뢰도의 향상**: 특정 시스템에 장애가 발생하더라도 다른 시스템이 대체하여 서비스를 지속할 수 있다.
- **처리기능의 분산**: 작업을 여러 컴퓨터에 나누어 처리함으로써 효율성과 유연성을 높인다.

시험에서 "부적절한 것"을 묻는 문제가 매우 반복적으로 출제되는데, **중앙집중형 처리**는 통신망의 주된 목적이 아니다. 통신망은 오히려 분산 처리를 지향하며, 중앙집중형 처리는 통신망이 보편화되기 이전의 방식이다.

**관련 문제**: (2015-19), (2016-22), (2017-19), (2018-22)

### 9.2 컴퓨터 통신망의 역사: ARPANET

**ARPANET**(Advanced Research Projects Agency Network)은 1969년 미국 국방부 산하 고등연구계획국(ARPA)에서 개발한 컴퓨터 통신망으로, **인터넷의 전신**이다. ARPANET은 **패킷 교환 방식**을 기반으로 데이터를 전송했으며, 이후 **TCP/IP 프로토콜**을 도입하여 인터넷 기술 발전의 중요한 토대가 되었다.

시험에서는 ARPANET과 관련된 키워드(인터넷의 전신, 패킷 교환 방식, TCP/IP 프로토콜)를 묻는 문제와, ARPANET과 관련이 가장 적은 것을 찾는 문제가 출제된다. **모스 부호**는 ARPANET과 관련이 가장 적다.

**관련 문제**: (2016-21), (2017-18), (2018-21), (2019-22)

### 9.3 데이터 교환 방식

데이터를 교환하는 방식은 크게 **회선 교환**, **메시지 교환**, **패킷 교환**으로 분류된다.

#### 9.3.1 회선 교환 방식

**회선 교환 방식**(circuit switching)은 데이터를 전송하기 전에 송신자와 수신자 사이에 **물리적인 전용 경로를 설정**하는 방식이다. 일단 회선이 설정되면 데이터는 중간 교환기에 저장되지 않고 설정된 회선을 통해 직접 전송된다. 따라서 교환기에 **저장장치가 없는** 교환 방식이다. 전화망이 회선 교환 방식의 대표적인 사례이다.

**관련 문제**: (2016-23), (2019-24)

#### 9.3.2 메시지 교환 방식

**메시지 교환 방식**(message switching)은 전송할 메시지 전체를 중간 교환기에 저장한 후 다음 교환기로 전달하는 **축적교환 방식**의 하나이다. 전용 경로를 설정하지 않고 메시지 단위로 전달하므로, 교환기에 저장장치가 필요하다.

#### 9.3.3 패킷 교환 방식

**패킷 교환 방식**(packet switching)은 메시지를 일정 크기의 작은 단위인 **패킷**으로 분할하여 전송하는 방식이다. 이 역시 축적교환 방식에 속하며, 패킷 교환 방식은 다시 **데이터그램 방식**과 **가상회선 방식**으로 나뉜다.

### 9.4 데이터그램 방식과 가상회선 방식

#### 9.4.1 데이터그램 방식

**데이터그램 방식**은 메시지를 일정 크기의 데이터그램으로 분할한 다음 각각의 데이터그램을 **독립적으로 전송**하는 비연결형(connectionless) 서비스이다. 각 데이터그램은 목적지 주소를 자체적으로 포함하고 있으며, 네트워크 내에서 서로 다른 경로를 통해 전송될 수 있고 도착 순서가 바뀔 수도 있다. 데이터그램 방식은 **축적교환 방식**이자 **패킷교환 방식**의 하나이며, 가상회선이라는 논리적 경로를 사용하지 않는다.

**관련 문제**: (2015-20), (2017-21), (2019-25)

#### 9.4.2 가상회선 방식

**가상회선 방식**은 데이터를 전송하기 전에 **가상회선**이라 부르는 논리적 경로를 먼저 설정한 후, 모든 패킷을 해당 경로를 통해 순서대로 전송하는 연결형(connection-oriented) 서비스이다. 연결 확립, 데이터 전송, 연결 해제의 3단계를 수행한다.

**관련 문제**: (2019-25)

### 핵심 키워드 요약

| 키워드 | 핵심 포인트 |
|--------|------------|
| 통신망 목적 | 자원 공유, 신뢰도 향상, 처리기능 분산 (중앙집중형 처리 X) |
| ARPANET | 인터넷의 전신, 패킷 교환 방식, TCP/IP 프로토콜 |
| 회선 교환 | 전용 경로 설정, 교환기에 저장장치 없음, 전화망 |
| 메시지 교환 | 메시지 전체 저장 후 전달, 축적교환 방식 |
| 패킷 교환 | 메시지를 패킷으로 분할 전송, 축적교환 방식 |
| 데이터그램 | 비연결형, 각 패킷 독립 전송, 가상회선 미사용 |
| 가상회선 | 연결형, 논리적 경로 설정, 순서대로 전송 |
