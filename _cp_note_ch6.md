## 제6장. 파일 구조와 데이터베이스

### 6.1 파일 구조와 인덱스

보조기억장치에 데이터를 저장할 때는 효율적인 검색과 관리를 위해 적절한 **파일 구조**를 선택해야 한다.

#### 6.1.1 순차적 파일 구조

**순차적 파일 구조**는 레코드를 저장된 순서대로 접근하는 방식이다. 전체 데이터를 처리할 때는 효율적이지만, 특정 레코드를 검색하려면 처음부터 순서대로 찾아야 하므로 검색 속도가 느리다.

#### 6.1.2 인덱스된 파일 구조

**인덱스된 파일 구조**는 **키 값과 레코드 주소**를 저장하는 별도의 인덱스(데이터 구조)를 사용하여 레코드를 빠르게 검색할 수 있는 구조이다. 인덱스를 만들면 전체 파일을 처음부터 끝까지 순차적으로 검색할 필요 없이, 인덱스를 통해 해당 레코드의 위치를 즉시 찾아갈 수 있으므로 **검색 시간이 크게 단축**된다.

**관련 문제**: (2015-10), (2019-11)

### 6.2 데이터베이스의 개념

**데이터베이스**(database)란 어떤 조직이나 단체를 유지하고 관리하는 데 필수적인 관련 데이터의 집합을 의미한다. 과거에는 응용프로그램 단위로 별도의 파일을 만들어 데이터를 개별 관리하는 **파일 시스템** 방식을 사용했으나, 이 방식은 데이터의 중복과 불일치 문제를 야기한다. 데이터베이스 시스템은 이러한 문제를 해결하기 위해 데이터를 **통합적**으로 관리하며, 여러 응용 프로그램이 동일한 데이터를 **공유**할 수 있도록 한다.

**관련 문제**: (2016-11)

### 6.3 데이터베이스 관리 시스템(DBMS)

**데이터베이스 관리 시스템**(DBMS, Database Management System)은 데이터베이스 내의 정보를 검색하거나, 정보를 저장하고 관리하기 위해 사용되는 소프트웨어 패키지이다. 데이터베이스 시스템에서 응용프로그램은 반드시 **DBMS를 통해서만** 데이터베이스에 접근한다.

DBMS의 필수 기능은 다음 세 가지로 분류된다.

- **데이터 정의 기능**(DDL): 데이터베이스의 구조를 정의한다.
- **데이터 조작 기능**(DML): 데이터의 검색, 갱신, 삽입, 삭제 등을 수행한다.
- **데이터 제어 기능**(DCL): 데이터베이스의 정확성과 안전성을 유지한다.

"검색 결과를 웹 화면에 정해진 서식으로 출력하는 화면 편집 기능"이나 "텍스트를 음성으로 출력하는 기능" 등은 DBMS의 필수 기능이 아니라 응용 프로그램이나 사용자 인터페이스 계층의 기능에 해당한다.

**관련 문제**: (2015-11), (2015-12), (2016-12), (2017-12), (2018-11), (2019-12)

### 6.4 데이터베이스 시스템의 장점

데이터베이스 시스템을 활용함으로써 얻을 수 있는 핵심적인 장점은 다음과 같다.

- **데이터의 일관성과 무결성 유지**: 여러 응용 프로그램이 동일한 데이터를 공유하면서도 데이터의 정확성과 신뢰성을 보장한다.
- **데이터 공유**: 통합적 구조를 통해 데이터를 공유할 수 있다.
- **데이터 중복 최소화**: 응용 프로그램별로 동일한 데이터를 중복 저장하지 않아도 된다.

한편, 데이터베이스 시스템은 정보시스템의 구축 및 운영 비용이 반드시 절감된다고 볼 수는 없으며, 비전문가가 쉽게 설계할 수 있는 것도 아니다. 또한 **데이터베이스 관리자**(DBA)가 데이터베이스의 관리, 운영, 통제 등의 역할을 담당한다.

**관련 문제**: (2015-13), (2016-11), (2017-13), (2018-12), (2019-13)

### 6.5 데이터 웨어하우스

**데이터 웨어하우스**(Data Warehouse)는 기업의 **의사 결정을 지원**하기 위해 여러 운영 시스템(마케팅, 판매, 회계 등)의 데이터베이스로부터 데이터를 추출하여 적절한 형태로 변환하고 통합 저장한 데이터의 집합체이다. 데이터 웨어하우스의 주요 특성은 네 가지로 요약된다.

- **주제 지향적**(Subject-Oriented): 의사 결정에 필요한 주제별로 구성된 데이터를 저장한다.
- **통합적**(Integrated): 여러 운영 시스템의 데이터를 통합한다.
- **시계열적**(Time-Variant): 정해진 시점별로 변화된 시계열 정보를 제공한다.
- **비휘발성**(Non-Volatile): 한 번 저장된 데이터는 업무 처리에 따라 지속적으로 갱신되지 않는다.

데이터 웨어하우스의 적절한 용도는 **분기별, 연도별 현황, 경향 분석** 등과 같이 의사 결정을 지원하는 데이터를 제공하는 것이며, 오늘 주문한 상품 목록을 조회하는 것과 같은 일상적인 업무 처리(OLTP)와는 목적이 다르다.

**관련 문제**: (2016-13), (2017-14), (2018-13), (2019-14)

### 핵심 키워드 요약

| 키워드 | 핵심 포인트 |
|--------|------------|
| 인덱스된 파일 구조 | 키 값+레코드 주소로 빠른 검색 |
| 데이터베이스 | 조직의 필수 관련 데이터 집합, 통합 관리 |
| DBMS | 정의(DDL)/조작(DML)/제어(DCL) 기능, 응용프로그램은 DBMS 통해서만 접근 |
| 데이터 일관성/무결성 | DB 시스템의 핵심 장점 |
| DBA | 데이터베이스 관리, 운영, 통제 담당 |
| 데이터 웨어하우스 | 의사 결정 지원, 주제 지향적, 시계열 정보, 비휘발성 |
