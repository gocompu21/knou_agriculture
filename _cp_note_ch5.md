## 제5장. 소프트웨어와 프로그래밍 언어

### 5.1 알고리즘

프로그램을 작성하기 위해서는 먼저 주어진 문제를 해결하기 위한 처리 절차를 설계해야 하는데, 이를 **알고리즘**(algorithm)이라 한다. 알고리즘이란 주어진 문제를 해결하기 위하여 **유한한 개수의 잘 정의된 명령으로 표현된 처리 절차**를 의미한다. 즉, 문제 해결에 필요한 단계들을 명확하고 체계적으로 나열한 것이 알고리즘이며, 컴퓨터 프로그램의 핵심적인 부분이다. 시험에서는 알고리즘의 정의를 묻는 문제가 매우 자주 출제되므로, "유한한 개수", "잘 정의된 명령", "처리 절차"라는 핵심 키워드를 반드시 기억해야 한다.

**관련 문제**: (2015-9), (2018-8), (2019-8)

### 5.2 프로그래밍 언어의 세대별 분류

프로그래밍 언어는 컴퓨터에게 작업을 지시하기 위해 사용하는 인공 언어이며, 발전 과정에 따라 세대별로 분류된다.

#### 5.2.1 제1세대 언어: 기계어

**기계어**는 0과 1의 이진수로 구성된 명령어 코드로서, CPU가 직접 이해하고 실행할 수 있는 유일한 언어이다. 별도의 번역 과정이 필요 없지만, 사람이 이해하고 작성하기가 매우 어렵다.

#### 5.2.2 제2세대 언어: 어셈블리어

**어셈블리어**는 각각의 기계어 명령을 사람이 이해하기 쉬운 기호(니모닉)로 표현한 **저급 언어**이다. 어셈블리어는 CPU의 종류에 따라 다르며, CPU가 직접 실행할 수 없으므로 **어셈블러**(assembler)라는 번역 프로그램이 필요하다.

#### 5.2.3 제3세대 언어: 고급 언어

**고급 언어**는 저급 언어에 비해 사람이 사용하는 자연어에 가까운 문장과 구문을 사용하는 프로그래밍 언어이다. C, Java, C++, Python 등이 대표적이며, 프로그래머의 생산성과 코드 가독성이 높다. 고급 언어로 작성된 프로그램은 CPU가 직접 실행할 수 없으므로, 반드시 **컴파일러**(compiler)나 **인터프리터**(interpreter)와 같은 **언어 번역기**가 필요하다.

**관련 문제**: (2016-7), (2017-9), (2019-9)

#### 5.2.4 제4세대 언어

**제4세대 언어**(4GL)는 사용자가 복잡한 프로그래밍 지식 없이도 특정 작업을 쉽게 수행할 수 있도록 설계된 언어이다. **리포트 생성**, **데이터 조작 및 분석** 등 특정 응용 분야에 최적화되어 있으며, 기존 순차적 고급 언어(3세대)에 비해 높은 수준의 기능을 제공하고 비절차적(Non-procedural) 특성을 가진다.

**관련 문제**: (2017-9)

### 5.3 원시 프로그램과 목적 프로그램

프로그래밍 언어로 작성된 프로그램을 **원시(source) 프로그램**이라 하고, 이를 컴퓨터가 실행할 수 있는 기계어 코드 형태로 변환한 프로그램을 **목적(object) 프로그램**이라 한다. 원시 프로그램을 목적 프로그램으로 변환하는 과정에서 언어 번역기가 사용된다.

#### 5.3.1 컴파일러와 인터프리터

**컴파일러**(compiler)는 원시 프로그램 전체를 한꺼번에 기계어 명령으로 구성된 목적 프로그램으로 번역하는 소프트웨어이다. 번역이 완료되면 독립적으로 실행 가능한 파일이 생성된다. 반면 **인터프리터**(interpreter)는 원시 프로그램을 한 줄씩 번역하면서 동시에 실행하는 방식으로, 별도의 목적 프로그램이 생성되지 않는다. **디버거**(debugger)는 프로그램의 오류를 찾아 수정하는 도구이므로 번역기와 구별해야 한다.

**관련 문제**: (2016-6), (2018-7)

### 5.4 소프트웨어의 사용권

소프트웨어는 사용권에 따라 다양한 형태로 분류된다.

#### 5.4.1 사유 소프트웨어

**사유 소프트웨어**(proprietary software)는 개발자나 기업이 저작권을 보유하며, 사용 및 배포에 제한을 두는 소프트웨어이다. 구매자는 라이선스 약관에 따라 사용할 수 있지만, 다른 사람과 자유롭게 공유하거나 소스 코드를 수정할 수 없다.

#### 5.4.2 프리웨어

**프리웨어**(freeware)는 소프트웨어 개발자가 저작권을 유지하면서 사용자에게 **사용권에 대한 비용 없이** 사용할 수 있도록 허용한 소프트웨어이다. 다만 프리웨어라고 해서 모든 상황에서 무료인 것은 아니며, **개인 사용자에게는 무료이지만 기관이나 기업에서는 사용료가 부과될 수 있으므로** 반드시 해당 소프트웨어의 라이선스 약관을 확인해야 한다. 또한 프리웨어는 소스 코드가 공개되지 않는 것이 일반적이므로, 자유롭게 분석하거나 수정할 수 있는 오픈 소스 소프트웨어와는 구별된다.

**관련 문제**: (2016-9), (2018-9)

#### 5.4.3 셰어웨어

**셰어웨어**(shareware)는 일정 기간 무료로 사용하게 하거나 일부 기능을 제한하여 배포한 후, 계속 사용하려면 정품을 구매하도록 유도하는 형태의 소프트웨어이다. 셰어웨어는 개발자가 저작권을 소유하고 있으며, **사유 소프트웨어**에 속한다. 자유 소프트웨어에 해당하지 않으며, 리눅스는 셰어웨어가 아니다.

**관련 문제**: (2019-10)

#### 5.4.4 자유 소프트웨어와 GNU 프로젝트

**자유 소프트웨어 운동**은 소프트웨어의 사용, 복사, 수정, 배포의 자유를 보장하는 것을 목표로 하며, 리처드 스톨만(Richard Stallman)이 시작한 **GNU 프로젝트**를 통해 구체화되었다. GNU 프로젝트는 유닉스와 유사하지만 자유 소프트웨어 원칙에 기반을 둔 운영체제를 만드는 것을 목표로 했다. **GNU GPL**(General Public License, 일반 공중 사용권)은 자유 소프트웨어 재단(FSF)에서 개발한 대표적인 라이선스로, 소프트웨어의 자유를 보장하며 **카피레프트**(Copyleft) 개념을 구현하고 있다.

**관련 문제**: (2016-8), (2017-10)

### 5.5 응용 소프트웨어

#### 5.5.1 스프레드시트

**스프레드시트**는 데이터를 행과 열로 구성된 **표 형식**으로 정리하고, 이를 기반으로 다양한 **계산이나 분석**, 차트 생성 등의 작업을 수행할 수 있도록 설계된 응용 소프트웨어이다. 마이크로소프트 엑셀(Excel)이 대표적이다.

**관련 문제**: (2017-8)

#### 5.5.2 ERP(전사적 자원관리)

**ERP**(Enterprise Resource Planning)는 기업의 **생산, 판매, 자재, 인사, 회계** 등 전반적인 업무 프로세스를 **하나의 체계로 통합**한 시스템이다. 기업의 자원을 효율적으로 운영하고 관리하기 위한 정보 시스템으로, 고객관계 관리(CRM)나 단순 데이터베이스와는 구별된다.

**관련 문제**: (2016-10), (2018-10)

### 핵심 키워드 요약

| 키워드 | 핵심 포인트 |
|--------|------------|
| 알고리즘 | 문제 해결을 위한 유한한 개수의 잘 정의된 명령의 처리 절차 |
| 고급 언어 (3세대) | 자연어에 가까운 문법, 컴파일러/인터프리터 필요 |
| 4세대 언어 | 리포트 생성, 데이터 분석 등 고수준 비절차적 기능 |
| 컴파일러 | 원시 프로그램 전체를 목적 프로그램으로 번역 |
| 인터프리터 | 한 줄씩 번역하면서 실행 |
| 프리웨어 | 무료 사용 허용, 개인/기관 조건 다를 수 있음 |
| 셰어웨어 | 체험판 배포 후 구매 유도, 사유 소프트웨어 |
| GNU GPL | 자유 소프트웨어 라이선스, 카피레프트 |
| ERP | 기업 전체 업무 프로세스 통합 관리 시스템 |
