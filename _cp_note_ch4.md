## 제4장. 운영체제

### 4.1 소프트웨어의 분류

컴퓨터에서 동작하는 소프트웨어는 크게 **시스템 소프트웨어**와 **응용 소프트웨어**로 분류된다. 시스템 소프트웨어는 컴퓨터 하드웨어를 제어하고 응용 프로그램이 실행될 수 있는 기반 환경을 제공하는 프로그램이다. 대표적으로 **운영체제**(Operating System)가 있으며, Windows 10, 리눅스(Linux), macOS 등이 이에 해당한다. 이 밖에 **언어번역기**(컴파일러, 인터프리터)나 장치 드라이버 등도 시스템 소프트웨어에 포함된다.

반면 **응용 소프트웨어**는 사용자가 특정 작업을 수행하기 위해 사용하는 프로그램이다. 워드프로세서, 스프레드시트(엑셀), 포토샵, 인터넷 브라우저, SNS 앱 등이 모두 응용 소프트웨어에 속한다. 시험에서는 시스템 소프트웨어와 응용 소프트웨어를 구별하는 문제가 자주 출제되므로, 운영체제가 시스템 소프트웨어라는 점을 확실히 기억해 두어야 한다.

**관련 문제**: (2015-6), (2017-6)

### 4.2 운영체제의 개념과 역할

**운영체제**(Operating System, OS)란 컴퓨터 시스템의 하드웨어와 소프트웨어 자원을 효율적으로 관리하고, 사용자에게 편리한 환경을 제공하는 핵심 시스템 소프트웨어이다. 운영체제가 수행하는 주요 역할은 다음과 같다.

- **자원 관리**: CPU, 메모리, 입출력 장치 등 하드웨어 자원을 효율적으로 배분하고 관리한다.
- **프로세스 관리**: 실행 중인 프로그램(프로세스)들이 원활하게 동작할 수 있도록 스케줄링하고 제어한다.
- **파일 시스템 관리**: 디스크에 저장된 파일의 생성, 삭제, 이름 변경, 복사, 이동 등을 관리한다.
- **사용자 인터페이스 제공**: 사용자가 컴퓨터와 상호작용할 수 있는 환경(GUI 또는 CLI)을 제공한다.
- **멀티태스킹 지원**: 여러 프로그램을 동시에 실행할 수 있도록 지원한다.

운영체제의 역할을 묻는 문제에서는 "문서 편집", "이메일 작성", "사진 업로드", "정보 검색" 등 **사용자가 직접 수행하는 작업**은 응용 소프트웨어의 영역이며, "하드웨어 관리", "파일 시스템 관리", "프로그램 실행 환경 제공", "사용자 인터페이스 제공" 등이 운영체제의 역할에 해당한다는 점에 유의해야 한다.

**관련 문제**: (2015-7), (2016-5), (2017-7), (2018-6), (2019-6)

### 4.3 운영체제의 종류

#### 4.3.1 유닉스(UNIX)

**유닉스**(UNIX)는 1969년 AT&T 벨연구소에서 개발된 운영체제로, **멀티태스킹**과 **멀티유저**를 지원하는 강력한 시스템이다. 개인용 워크스테이션부터 중대형 컴퓨터에 이르기까지 광범위한 시스템에서 사용되며, 현대의 여러 운영체제에 큰 영향을 미쳤다. 특히 애플의 **OS X(macOS)**는 유닉스 기반으로 만들어졌으며, 리눅스 역시 유닉스의 설계 철학을 계승하여 개발되었다.

**관련 문제**: (2019-7)

#### 4.3.2 리눅스(Linux)

**리눅스**(Linux)는 1991년 리누스 토르발스(Linus Torvalds)가 개발을 시작한 운영체제로, **자유 소프트웨어** 운동과 **오픈 소스** 철학에 입각하여 만들어졌다. 소스 코드가 완전히 공개되어 있으므로 누구나 자유롭게 사용, 수정, 배포할 수 있다. 서버 시스템, 임베디드 기기, 모바일(안드로이드) 등 다양한 분야에서 널리 활용되고 있다. 시험에서는 오픈 소스 운영체제를 묻는 문제에서 리눅스가 정답으로 출제되는 경우가 많으므로, Windows나 macOS(OS X), iOS와 구별하여 기억해야 한다.

**관련 문제**: (2015-8)

#### 4.3.3 Windows

마이크로소프트가 개발한 **Windows**는 개인용 컴퓨터에서 가장 널리 사용되는 운영체제이다. GUI(그래픽 사용자 인터페이스)를 기반으로 하며, Windows 8, Windows 10 등의 버전이 출시되어 왔다. Windows는 상용 소프트웨어로, 소스 코드가 공개되지 않은 **비공개(proprietary) 소프트웨어**에 해당한다.

### 4.4 파일과 파일 시스템

**파일**(file)이란 보조기억장치(하드디스크, SSD 등)에 저장된 관련 데이터의 집합을 말한다. 파일은 텍스트뿐만 아니라 이미지, 음성, 동영상, 실행 프로그램 등 다양한 형태의 데이터를 저장할 수 있다. 파일이 저장되는 곳은 주기억장치가 아닌 **보조기억장치**라는 점에 유의해야 한다.

운영체제의 **파일 시스템**(file system)은 파일을 효율적으로 관리하기 위한 체계로서, 파일의 생성, 삭제, 이름 변경, 복사, 이동 등 다양한 관리 기능을 제공한다. 파일 접근 방식에 따라 **순차파일**(저장된 순서대로 접근)과 **직접파일**(임의 위치에 직접 접근)로 구분되는데, 직접파일은 저장된 순서와 무관하게 원하는 레코드에 바로 접근할 수 있는 구조이다.

**관련 문제**: (2017-11)

### 핵심 키워드 요약

| 키워드 | 핵심 포인트 |
|--------|------------|
| 시스템 소프트웨어 | 하드웨어 제어, 응용 프로그램 실행 환경 제공 (예: 운영체제, 언어번역기) |
| 응용 소프트웨어 | 사용자 작업 수행 (예: 워드프로세서, 엑셀, 포토샵) |
| 운영체제 역할 | 자원 관리, 프로세스 관리, 파일 시스템 관리, UI 제공, 멀티태스킹 |
| 유닉스(UNIX) | 멀티태스킹/멀티유저, 워크스테이션~대형, macOS 기반 |
| 리눅스(Linux) | 오픈 소스, 자유 소프트웨어, 소스 코드 공개 |
| Windows | 개인용 PC 대표, 상용 비공개 소프트웨어 |
| 파일 시스템 | 파일 생성/삭제/복사/이동 관리, 보조기억장치에 저장 |
| 순차파일 vs 직접파일 | 순서대로 접근 vs 임의 위치 직접 접근 |
