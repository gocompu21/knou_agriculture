## 제10장. 컴퓨터 네트워크

### 10.1 통신망의 규모에 따른 분류

컴퓨터 통신망은 커버하는 지리적 범위에 따라 다음과 같이 분류된다. 규모가 작은 것부터 큰 순서로 나열하면 **PAN < LAN < MAN < WAN**이다.

- **PAN**(Personal Area Network): 개인 영역 통신망. 블루투스 등 개인 기기 간 연결.
- **LAN**(Local Area Network): 근거리 통신망. 한 건물이나 캠퍼스 내의 장치들을 연결하며, **규모가 가장 작은** 통신망이다.
- **MAN**(Metropolitan Area Network): 도시권 통신망. 하나의 도시 범위를 커버한다.
- **WAN**(Wide Area Network): 광역 통신망. 국가 또는 대륙을 넘어서는 광대한 지역을 연결하며, **규모가 가장 큰** 통신망이다.

이와 별도로 **VAN**(Value-Added Network, 부가가치 통신망)이 있는데, 이는 통신망의 물리적 거리에 따른 분류가 아니라 **제공하는 서비스의 종류**에 따른 분류이다. VAN은 공중전기통신업자로부터 전용선을 임차하고 여기에 컴퓨터를 접속하여 구성하는 통신망으로, **홈뱅킹, 홈쇼핑** 등 정보의 부가가치를 창출하는 서비스를 제공한다.

**관련 문제**: (2015-21), (2016-24), (2017-22), (2018-23), (2019-26)

### 10.2 네트워크 연결 장치

컴퓨터 통신망에서 네트워크를 연결하고 데이터를 전달하는 데 사용되는 주요 장치는 다음과 같다.

#### 10.2.1 허브(Hub)

**허브**(hub)는 하나의 통신 노드에서 수신한 신호를 정확히 재생하여 연결된 **다른 모든 노드**로 전송하는 것이 주 기능인 장치이다. OSI 모델의 물리 계층(Layer 1)에서 동작하며, 특정 목적지로 트래픽을 필터링하거나 경로를 선택하는 기능은 없다. 리피터(repeater)와 유사한 신호 증폭 기능을 수행한다.

**관련 문제**: (2017-23), (2019-27)

#### 10.2.2 브리지(Bridge)

**브리지**(bridge)는 동일한 프로토콜을 사용하는 두 개의 네트워크 세그먼트를 연결하는 장치이다. MAC 주소를 기반으로 데이터를 필터링하여 필요한 세그먼트에만 전달한다.

#### 10.2.3 라우터(Router)

**라우터**(router)는 서로 다른 네트워크를 연결하고, 데이터 패킷의 최적 경로를 결정하여 전달하는 장치이다. IP 주소를 기반으로 라우팅을 수행한다.

#### 10.2.4 게이트웨이(Gateway)

**게이트웨이**(gateway)는 **서로 다른 통신 프로토콜을 사용하는** 여러 개의 네트워크들을 서로 연결해 주는 장치이다. 프로토콜을 변환하여 이기종 네트워크 간의 통신을 가능하게 한다. 시험에서 "서로 다른 통신 프로토콜을 사용하는 네트워크들을 연결"하는 장치를 묻는 문제가 반복 출제되므로, 게이트웨이를 확실히 기억해야 한다.

참고로 **멀티드롭**(multidrop)은 통신 회선을 공유하는 연결 방식을 의미하는 용어이며, 네트워크 연결 장치 자체를 지칭하는 것이 아니다.

**관련 문제**: (2015-22), (2016-25), (2018-25)

### 10.3 네트워크 토폴로지(형태)

네트워크를 구성하는 물리적 또는 논리적 배치 형태를 **토폴로지**(topology)라 한다. 주요 네트워크 토폴로지는 다음과 같다.

#### 10.3.1 성형(Star) 네트워크

**성형 네트워크**는 모든 노드가 **중앙의 허브나 스위치**에 직접 연결되는 구조이다. 노드 추가나 제거가 쉽고 개별 노드의 장애가 다른 노드에 영향을 주지 않지만, **중앙 컴퓨터(허브)에 고장이 발생하면 전체 네트워크가 마비**된다는 치명적인 약점이 있다.

**관련 문제**: (2015-23), (2017-27)

#### 10.3.2 환형(Ring) 네트워크

**환형 네트워크**는 각 노드가 양쪽의 인접한 노드와 연결되어 고리 형태를 이루는 구조이다. 데이터는 한 방향 또는 양방향으로 순환하며 전달된다. **이중 링**(dual ring) 구성의 경우 두 개의 경로가 있어, 한 경로에 문제가 발생하면 **다른 경로를 예비용**으로 사용하여 통신을 계속할 수 있다.

**관련 문제**: (2017-27), (2018-27)

#### 10.3.3 버스형(Bus) 네트워크

**버스형 네트워크**는 모든 단말기가 **하나의 공용 선로(버스)**에 연결되어 그 선로를 공용으로 사용하는 구조이다. 한 순간에 **하나의 단말기만이 데이터를 전송**할 수 있으며, 케이블 설치가 간단하고 비용이 적게 든다.

**관련 문제**: (2019-31)

#### 10.3.4 망형(Mesh) 네트워크

**망형 네트워크**는 각 노드가 다른 모든 또는 대부분의 노드와 직접 연결되는 구조이다. 통신선로의 수가 많아 특정 경로에 장애가 발생하더라도 다른 경로를 통해 통신이 가능하므로, **신뢰성(reliability)이 가장 높다**. 다만 많은 수의 통신선로가 필요하여 비용이 높다.

**관련 문제**: (2017-27)

### 핵심 키워드 요약

| 키워드 | 핵심 포인트 |
|--------|------------|
| PAN < LAN < MAN < WAN | 규모 순서, WAN이 가장 큼, LAN이 거리 분류 중 가장 작음 |
| VAN | 부가가치 통신망, 거리 기반 분류 아님, 홈뱅킹/홈쇼핑 |
| 허브 | 신호 재생 후 모든 노드로 전송, 물리 계층 |
| 게이트웨이 | 서로 다른 프로토콜 네트워크 연결, 프로토콜 변환 |
| 성형 네트워크 | 중앙 허브 집중, 중앙 고장 시 전체 마비 |
| 환형 네트워크 | 이중 링 시 예비 경로, 고장 시 우회 가능 |
| 버스형 네트워크 | 공용 선로, 한 순간에 하나만 전송 |
| 망형 네트워크 | 신뢰성 최고, 다중 경로, 비용 높음 |
