## 제11장. 인터넷과 웹

### 11.1 인터넷의 개념

**인터넷**(Internet)은 "Interconnected network"에서 유래한 용어로, 전 세계의 컴퓨터 네트워크들이 상호 연결된 거대한 통신망이다. 미국 국방용 네트워크인 **ARPANET**이 전신이며, 1983년에 **TCP/IP 프로토콜**을 바탕으로 본격적으로 발전하였다.

**관련 문제**: (2015-26), (2016-28), (2018-28)

### 11.2 인터넷의 특성

인터넷의 주요 특성은 다음과 같다.

- **개방적인 구조**: 누구나 네트워크에 참여하고 정보를 공유할 수 있다.
- **호스트간의 평등성**: 인터넷에 연결된 모든 컴퓨터는 정보의 송수신에서 평등하다.
- **독자적인 주소 할당**: 인터넷에 연결되는 모든 컴퓨터는 **고유한 IP 주소**를 할당받으며, 다른 컴퓨터와 주소를 공유하지 않는다.
- **분산형 구조**: 인터넷은 중앙 집중형 구조가 아니라 분산형 구조이다. 특정 중앙 기관이 모든 통신을 제어하지 않으며, 네트워크의 일부가 손상되어도 다른 경로를 통해 통신을 유지할 수 있다.

시험에서 부적절한 특성을 찾는 문제가 반복 출제되는데, "신뢰성 있는 보안성", "중앙 집중형 구조", "세션 주소" 등이 인터넷의 특성으로 부적절한 항목이다.

**관련 문제**: (2015-27), (2016-29), (2017-28), (2019-32)

### 11.3 IP 주소와 도메인 이름

#### 11.3.1 IP 주소

**IP 주소**(Internet Protocol Address)는 인터넷에 연결된 각 컴퓨터를 식별하기 위한 고유한 숫자 주소이다. **IP**(Internet Protocol)는 **비연결형** 데이터 전송을 위한 프로토콜로, 연결 설정 과정 없이 독립적인 데이터그램을 전송한다.

**관련 문제**: (2015-30)

#### 11.3.2 도메인 이름

**도메인 이름**(Domain Name)은 숫자로 구성된 IP 주소 대신에 의미가 있는 단어를 사용해 네트워크에 일종의 별명을 붙여서 사용하는 것이다. 예를 들어 "knou.ac.kr"과 같이 기억하기 쉬운 문자열로 표현한다.

도메인 이름의 체계에서 사용되는 주요 식별 코드는 다음과 같다.

- **ac**: **교육기관** (academic)
- **or**: **공공기관** (organization)
- **co**: 상업기관 (company)
- **go**: 정부기관 (government)

**관련 문제**: (2016-30), (2017-29), (2018-29), (2018-30)

### 11.4 URL(Uniform Resource Locator)

**URL**(Uniform Resource Locator)은 인터넷에서 접근 가능한 **특정 자원의 주소**를 일관되게 표현할 수 있는 형식이다. URL의 구성 요소에는 **프로토콜 이름**, **도메인 이름**, **컴퓨터 파일의 계층적 위치 정보** 등이 포함된다. "인터넷 주소"라는 용어는 URL의 직접적인 구성 요소로 보기 어렵다.

**관련 문제**: (2015-28), (2016-31), (2017-30), (2018-31)

### 11.5 웹(WWW) 서비스와 HTTP

**WWW**(World Wide Web) 서비스는 인터넷상에서 하이퍼텍스트 기반의 문서를 주고받는 서비스이다. 웹 서비스를 사용할 때 필요한 통신 프로토콜은 **HTTP**(HyperText Transfer Protocol)이다. 웹 브라우저가 웹 서버에 웹 페이지를 요청하고, 웹 서버가 요청된 페이지를 브라우저로 전송할 때 HTTP가 사용된다. **HTML**은 웹 페이지를 작성하기 위한 마크업 언어이지 프로토콜이 아니므로 구별해야 한다.

**관련 문제**: (2015-29), (2016-32), (2017-31)

### 11.6 이메일 관련 프로토콜

이메일을 송수신할 때 사용되는 통신 프로토콜은 다음과 같다.

- **SMTP**(Simple Mail Transfer Protocol): 이메일 **발송**에 사용된다.
- **POP3**(Post Office Protocol 3): 이메일 **수신**에 사용되며, 서버에서 메일을 다운로드한다.
- **IMAP**(Internet Message Access Protocol): 이메일 **수신**에 사용되며, 서버에서 메일을 관리한다.

**TCP**(Transmission Control Protocol)는 이들 프로토콜이 안정적으로 데이터를 전송할 수 있도록 하는 전송 계층 프로토콜이므로, 이메일 송수신을 직접 담당하는 프로토콜과는 구별된다.

**관련 문제**: (2019-30)

### 11.7 인터넷 서비스와 웹 호스팅

인터넷을 통해 제공되는 다양한 서비스에는 **전자상거래**, **이러닝**(e-Learning), **IPTV**, 정보 검색, 원격 진료 등이 있다. 한국방송통신대학교에서 제공하는 인터넷 강의는 **이러닝**의 대표적인 사례이다. 컴파일러와 같은 시스템 소프트웨어는 인터넷 응용 서비스에 해당하지 않는다.

**웹 호스팅**(Web Hosting)은 웹사이트들을 위한 홈페이지 운영에 필요한 기억 공간, FTP 서비스, 메일 서버 등을 위한 저장 공간을 제공하고 유지해 주는 서비스이다. 웹 서버를 직접 구축하기 어려운 개인이나 기업이 전문 업체로부터 임대하여 사용한다.

**관련 문제**: (2015-31), (2015-32), (2017-32)

### 11.8 ENIAC

**ENIAC**은 1946년에 개발된 최초의 전자식 범용 컴퓨터 중 하나로, 컴퓨터 자체의 발전에는 중요한 역할을 했지만 **인터넷과는 관련이 가장 적다**. 인터넷의 개념이 등장하기 훨씬 이전에 개발되었으며, 네트워크 기술과는 직접적인 관련이 없다.

**관련 문제**: (2016-28), (2018-28)

|--------|------------|
| 인터넷 | Interconnected network, ARPANET 전신, TCP/IP 기반 |
| 인터넷 특성 | 개방 구조, 호스트 평등, 고유 IP 주소, 분산형 (중앙집중 X) |
| IP | 비연결형 프로토콜, 인터넷 계층 |
| 도메인 이름 | IP 주소의 문자열 별명, ac=교육, or=공공, co=상업, go=정부 |
| URL | 인터넷 자원의 일관된 주소 표현 형식 |
| HTTP | 웹(WWW) 서비스 프로토콜 |
| SMTP/POP3/IMAP | 이메일 송수신 프로토콜 |
| 웹 호스팅 | 홈페이지 공간/서버 제공 서비스 |
| 이러닝 | 인터넷 기반 교육 서비스, 방송대 강의 |
