## 제12장. 정보보안

### 12.1 정보보안의 목표

**정보보안**의 3가지 핵심 목표는 **CIA 트라이어드**로 불리며 다음과 같다.

- **기밀성**(Confidentiality): 인가된 사용자만이 정보에 접근할 수 있도록 보장한다.
- **무결성**(Integrity): 정보가 인가되지 않은 방법으로 변경되거나 훼손되지 않도록 보장한다.
- **가용성**(Availability): 인가된 사용자가 필요할 때 정보에 접근할 수 있도록 보장한다.

시험에서는 "정보보안의 3가지 목표로 부적절한 것"을 묻는 문제가 출제되는데, **개방성**은 정보보안의 목표에 해당하지 않는다.

**관련 문제**: (2018-32)

### 12.2 네트워크 보안 공격 유형

네트워크 환경에서 제3자에 의한 불법적인 공격은 다음과 같이 분류된다.

- **방해**(Interruption): 정보 자원을 사용할 수 없게 만드는 공격 (가용성 침해)
- **가로채기**(Interception): 비인가자가 정보를 도청하는 공격 (기밀성 침해)
- **변조**(Modification): 정보를 불법으로 수정하는 공격 (무결성 침해)
- **위조**(Fabrication): 거짓 정보를 만들어 전송하는 공격 (인증 침해)

**부정**(fraud)은 범죄 행위의 한 유형이지만, 네트워크 보안 공격의 일반적인 분류에는 포함되지 않는다.

**관련 문제**: (2015-33)

### 12.3 해킹 기법

해킹에 사용되는 주요 방법은 다음과 같다.

- **백도어**(Backdoor): 시스템에 비밀리에 설치된 통로로, 정상적인 인증 절차를 우회하여 시스템에 접근할 수 있게 한다.
- **스니핑**(Sniffing): 네트워크를 통해 전송되는 데이터를 **도청**하는 행위이다.
- **스푸핑**(Spoofing): 송신자의 주소나 신원을 **위조**하여 다른 사용자나 시스템으로 가장하는 행위이다.
- **DDoS**(Distributed Denial of Service): 다수의 컴퓨터에서 동시에 특정 서버에 대량의 트래픽을 보내 서비스를 마비시키는 분산 서비스 거부 공격이다.

해킹 방법과 관련이 가장 적은 것으로는 **백신**(vaccine)과 **인코딩**(encoding)이 출제된다. 백신은 해킹을 **방어**하는 보안 소프트웨어이며, 인코딩은 데이터 형식을 변환하는 일반적인 기술이다.

**관련 문제**: (2016-33), (2017-33), (2018-33), (2019-33)

### 12.4 악성 프로그램

컴퓨터 시스템에 해를 끼치는 악성 프로그램의 주요 유형은 다음과 같다.

- **컴퓨터 바이러스**(Virus): 다른 프로그램에 기생하여 자기 자신을 복제하고 전파하는 악성 코드이다.
- **컴퓨터 웜**(Worm): 네트워크를 통해 스스로 전파되는 악성 프로그램으로, 다른 프로그램에 기생하지 않고 독립적으로 실행된다.
- **트로이 목마**(Trojan Horse): 유틸리티 프로그램과 같이 좋은 기능의 프로그램처럼 **위장**하여 배포하지만, 그 안에 내장된 악성 코드를 이용하여 사용자의 정보를 **탈취**하기 위해 만들어진 악성 프로그램이다.

**백신**(vaccine)은 이러한 악성 프로그램을 탐지하고 제거하여 시스템을 보호하는 소프트웨어이므로, 악성 프로그램에 해당하지 않는다.

**관련 문제**: (2016-35), (2018-33)

### 12.5 암호화

**암호화**(encryption)란 **평문**(plaintext)을 제3자가 알아볼 수 없는 **암호문**(ciphertext)으로 변환하는 과정을 말한다. 반대로 암호문을 다시 평문으로 변환하는 과정은 **복호화**(decryption)라 한다. 인코딩은 암호화나 복호화와는 다른 개념이다.

#### 12.5.1 대칭키 암호화 방식

**대칭키 암호화 방식**(비밀키 암호화, 공통키 암호화)은 암호화 키와 복호화 키가 **서로 같은** 암호화 방식이다. 하나의 비밀 키를 송신자와 수신자가 공유하여 사용한다. 대표적인 대칭키 암호화 알고리즘으로는 **DES**(Data Encryption Standard)가 있다. DES는 미국 국립표준기술원(NIST)에 의해 연방 표준으로 채택된 대칭키 암호화 알고리즘이지, 공개키 방식이 아니며 MIT대학의 표준도 아니다.

**관련 문제**: (2015-34), (2016-34), (2017-35), (2018-34)

#### 12.5.2 공개키 암호화 방식

**공개키 암호화 방식**(비대칭키 암호화)은 암호화 키와 복호화 키가 **서로 다른** 암호화 방식이다. **암호화 키(공개키)**는 모두에게 공개하지만, **복호화 키(개인키)**는 본인만 보관하여 비밀로 유지한다. 대표적인 공개키 암호화 알고리즘으로는 **RSA**가 있다.

**관련 문제**: (2019-34)

### 12.6 네트워크 보안 강화 방법

네트워크의 보안을 강화하기 위한 주요 방법은 다음과 같다.

- **암호화**: 데이터를 암호문으로 변환하여 도청에 대비한다.
- **방화벽**(Firewall): 네트워크와 네트워크 사이에서 송수신되는 패킷을 검사하여 **조건에 맞는 패킷들만 통과**시키는 소프트웨어나 하드웨어를 총칭한다. 미리 정의된 보안 규칙에 따라 네트워크 트래픽을 허용하거나 차단한다.
- **전자 서명**(Digital Signature): 전자 문서의 작성자를 확인하고 문서의 무결성을 보장한다.

**서비스 거부**(Denial of Service, DoS)는 보안 강화 방법이 아니라, 네트워크를 마비시키는 **공격 기법**이다. 시험에서 "보안 강화 방법으로 부적절한 것"을 묻는 문제가 반복 출제되므로, 서비스 거부가 공격이라는 점을 확실히 기억해야 한다.

**관련 문제**: (2015-35), (2017-34), (2018-35), (2019-35)

### 핵심 키워드 요약

| 키워드 | 핵심 포인트 |
|--------|------------|
| CIA 트라이어드 | 기밀성, 무결성, 가용성 (개방성 X) |
| 보안 공격 유형 | 방해, 가로채기, 변조, 위조 (부정 X) |
| 스니핑 | 네트워크 도청 |
| 스푸핑 | 주소/신원 위조 |
| 트로이 목마 | 유용한 프로그램으로 위장한 악성 코드 |
| 백신 | 악성 프로그램 탐지/제거 보안 소프트웨어 (해킹 방법 X) |
| 대칭키 암호화 | 암호화키 = 복호화키, 대표: DES |
| 공개키 암호화 | 암호화키(공개) ≠ 복호화키(비공개), 대표: RSA |
| 방화벽 | 패킷 검사 후 조건에 맞는 것만 통과 |
| 서비스 거부(DoS) | 공격 기법, 보안 강화 방법 아님 |
